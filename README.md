# Learn by Phase

A structured, interactive learning platform for full-stack developers who want to go beyond syntax and actually understand how things work.

Live at **[learn-by-phase.vercel.app](https://learn-by-phase.vercel.app)**

---

## Overview

Learn by Phase is a self-paced learning system organized into 5 progressive phases — JavaScript, TypeScript, Backend Architecture, Networking & Security, and DSA. Each phase contains multiple concepts, and every concept is broken down the same way so learning feels consistent and intentional.

The idea is simple: most developers learn *what* something is but never learn *why* it exists or *how* it actually works under the hood. This platform fixes that.

---

## How Each Concept Is Structured

Every single concept across all 5 phases follows the same six-layer breakdown:

**Why It Exists** — The reason this concept was invented. You understand the problem it solves before you touch any code.

**Mental Model** — An analogy that makes the internals click. Something that sticks.

**Deep Dive** — How it actually works under the hood. Not surface level.

**Code** — A real, production-level implementation — not a toy example stripped of all nuance.

**Bugs** — The actual mistakes people make with this concept in production, and why they happen. Understanding the failure mode teaches more than the fix alone.

**Challenge** — Something to build yourself from scratch. This is the part that converts reading into knowing.

---

## The Five Phases

**Phase 01 — JavaScript Foundations**
The WHY layer. Covers execution context, closures, the event loop, prototypes, `this` binding, promises, async/await, and higher order functions. Everything that runs silently beneath every framework you've ever used.

**Phase 02 — TypeScript**
Type thinking, not just type syntax. Covers generics, utility types, mapped types, conditional types, type guards, and how TypeScript's type system actually works at a structural level.

**Phase 03 — Backend Architecture**
Scalable Node.js patterns. Covers layered architecture, SOLID principles, dependency injection, database indexing, Redis caching, message queues, REST design, authentication, and error handling.

**Phase 04 — Networking & Security**
How the internet works — and how it gets broken. Covers the full HTTP stack, TLS internals, JWT and session auth, the OWASP Top 10, rate limiting, and infrastructure security.

**Phase 05 — DSA Pro**
Algorithmic thinking applied to real systems. Covers Big O analysis, arrays and hash maps, linked lists, binary search, trees, graphs, dynamic programming, heaps, sorting, and how these patterns show up in actual backend code.

## Who This Is For

Developers who already know how to build things but want to understand what they're building at a deeper level. The kind of person who has used `async/await` a hundred times but isn't completely sure what the event loop is doing, or who has written JWTs but never read the spec.
